# Self-Review Checklist

Before opening a pull request, I confirm that:

- [ ] **Code correctness**: Does the code accurately do what the title says and solve the intended problem?
- [ ] **Test coverage**: Have I included tests for new logic, and do all existing tests pass?
- [ ] **Documentation**: Is the README updated to reflect any changes in behavior or setup?
- [ ] **Scope**: Is this PR limited to one logical change, avoiding "scope creep"?
- [ ] **No debug artifacts**: Have I removed all `breakpoint()`, temporary `print()` statements, and commented-out code? 
- [ ] **No unnecessary files were added**
- [ ] **Code follows project style and conventions**
- [ ] **Changes were tested locally before creating the PR**
